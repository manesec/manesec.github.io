https://api.vulnlab.com/api/v1/share?id=4b25f275-07c8-41fa-8357-6f29508f2057

首先使用了 nmap 掃描技術來發現目標系統上開放的端口，包括 512 (Rexec)、513 (Rlogin)、514 (Rsh) 和 873 (Rsync)，以及運行在 3000 端口上的 Gitea 應用。這些端口的開放表明該系統可能允許無密碼的遠程訪問，但需要綁定 DNS 域名。接著，通過 Rsync 枚舉發現了一個名為 backups 的文件夾並下載其內容，其中搜索到可能包含密碼的文件，但部分文件如 jar 和 js 文件被排除在外。隨後，發現了一個 hash 和一個用戶名，嘗試破解 hash 未果，轉而關注另一個文件中可能的加密密碼。利用工具 pwn_jenkins 並參考官方文檔，解密了加密的密碼，但未能成功登錄 SSH。此時，注意到 Gitea 應用中存在一個 buildadm 用戶及其相關項目，通過修改 Jenkins 構建項目腳本獲得了一個 shell。在用戶目錄中發現 .rhosts 文件，顯示兩個允許無密碼 Rsh 登錄的域名，進一步調查發現可通過 DNS 修改來利用這一點。使用 ligolo 建立 tunnel，掃描內部網絡以訪問 DNS 修改界面，並成功在 powerdnsadmin 數據庫中查找用戶信息，破解並獲得 admin 密碼。最後，通過修改 DNS 設置，成功利用 Rsh 無密碼登錄獲得 root 權限。

Nmap scan

從nmap的結果中可以看到開了 512 (Rexec)​，513 (Rlogin)​ 和 514 (Rsh)​ ，這三個都是用與遠程訪問而且可以免密碼，但是需要綁定 dns 域名。

另外注意到 873 (Rsync)​ 也開啓了，還有一個 gitea 在 3000 端口。
