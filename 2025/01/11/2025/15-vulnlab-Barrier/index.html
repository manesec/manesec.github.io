VulnLab - Machine - Barrier

manesec
2025-01-11

​​

https://api.vulnlab.com/api/v1/share?id=8ffa1a3b-cbd4-41ce-8d8d-ca53e59ef094

首先，發現了一個公開的 GitLab 項目，通過查看其提交歷史，獲得了用戶 satoru 的賬號和密碼。接著，利用這些憑證成功登錄到一個運行在 9000 端口的 authentik 服務，這是一個單一登入（SSO）系統。通過這個系統，進一步進入了 GitLab，並發現了其版本信息。利用這些信息，搜索到了相關的漏洞 CVE-2024-45409，這是一個涉及 Ruby SAML 庫的漏洞，允許攻擊者偽造任意用戶身份。為了利用這個漏洞，嘗試偽造 GitLab 管理員 akadmin 的身份，並成功登錄。進入後，發現 GitLab 部署了一個暫停的 gitlab-runner，通過啟動它並創建一個自動化腳本，獲得了一個反向 shell，從而進一步探索系統內部。接下來，發現了一個管理員權限的 AUTHENTIK_TOKEN，並利用它修改了 authentik 管理員的密碼，成功進入後台。最終，通過分析 Guacamole 的配置文件，找到了 maki_adm 用戶的 SSH 密鑰，並成功登錄，獲得了更高的系統權限。

Exploit CVE-2024-45409 and login as akadmin

既然有了版本就可以嘗試搜索一下相關的 exploit，由於登錄是經過 authentik​ 也就是 SSO ，所以關鍵字會嘗試搜索 SSO​：

​​

結果看到了：

  • https://about.gitlab.com/releases/2024/09/17/patch-release-gitlab-17-3-3-released/

  • https://github.com/synacktiv/CVE-2024-45409

看起來這個關於是 gitlab 的漏洞，準確來説是 Ruby SAML library​ ：

This exploit injects the DigestValue​ of the modified assertion into the StatusDetail​ element, allowing it to smuggle the XPath selector that will use this value instead of the one in the SignedInfo​ block.

https://github.com/synacktiv/CVE-2024-45409

通過描述也就是使用 SSO 進行登錄的時候，可以利用 CVE-2024-45409​ 來僞造任意用戶。

Logout and Using SSO login again and intercept SAMLResponse

首先先登出：

​​

在使用 SSO 進行登錄，由於登錄會經過 authentik​ ，而驗證身份也會在 authentik​。儅驗證通過後 authentik​ 會通過 url 的 SAMLResponse​ 傳遞簽名后的 payload 給 gitlab，gitlab會嘗試核對這段 payload 有沒有被進行修改（也就是 <SignedInfo>​）。核對完成后沒有被修改就會以 payload 裏面的用戶名進行登錄。

但如果 gitlab 核對這段payload時的程序有bug，如果能找到一種方法就可以繞過核對，這樣就可以以任意用戶登錄。（這就是 CVE-2024-45409​ 的原理）

所以如果使用 burp 進行攔截請求，可以看到authentik​ 驗證完成后會將用戶名簽名打包並前面進去xml，然後通過SAMLResponse​傳遞給 github：

​​

從 HTTP 頭文件看到這段 payload 經過 flate​ 壓縮：

​​

所以可以使用 cyberchef 進行base64解碼后再解壓：

​​

Exploiting CVE-2024-45409
