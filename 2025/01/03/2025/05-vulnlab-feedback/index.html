manesec
2025-01-03

​​

https://api.vulnlab.com/api/v1/share?id=091076ed-04df-46c6-91b4-95fc71313d19

首先通過端口掃描發現目標系統僅開放 8080 端口，並確認其運行的是 Apache Tomcat 9.0.56 版本。接著，針對該版本進行漏洞搜索，發現了與默認密碼相關的爆破方法，並使用 hydra 工具進行嘗試，最終獲得了一組賬號密碼。然而，登錄後臺管理頁面時並未要求登錄，這引起了懷疑。進一步檢查發現，該版本的 Tomcat 在 2021 年曾爆出知名的 log4j 漏洞，於是決定掃描其子目錄以尋找可利用的入口。在調查中，發現了一個名為 feedback 的功能，並嘗試利用 log4j 的 payload 進行攻擊，結果獲得了回顯，證明該漏洞可被觸發。隨後，根據相關的利用代碼（PoC），下載了特定版本的 Java 開發工具包，並按照指示準備好環境。最終，通過提交特定的 JNDI 請求，成功獲得了 shell 訪問權限。為了進一步穩定控制，計劃上傳一個 C2 服務器。隨後，檢查 Tomcat 的配置文件，發現後臺賬號密碼存儲在 config/tomcat-users.xml 文件中，最終成功獲得 root 密碼。
