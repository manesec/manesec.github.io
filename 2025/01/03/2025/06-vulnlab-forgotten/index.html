VulnLab - Machine - Forgotten

manesec
2025-01-03

​​

https://api.vulnlab.com/api/v1/share?id=73d888d6-d00f-47d8-928e-3f887f470289

首先針對一台僅開放 80 端口的服務器進行了初步的探查，發現進入後顯示為 Forbidden，這引起了進一步的關注。隨後進行了掃描，發現該服務器上有一個調查（survey）功能，並且進入安裝向導時，驚訝地發現並不需要輸入安裝密碼。這使得測試者決定自行搭建一個數據庫環境，以便進行後續的操作，選擇了使用 Docker 來快速啟動 MySQL 數據庫，並根據 Docker 的指南進行配置。當數據庫搭建完成後，測試者在安裝向導中填寫了管理員密碼，並在接下來的步驟中成功登錄到後臺。這時，測試者發現可以上傳插件，這為後續的攻擊提供了機會。於是，測試者準備了一個惡意插件，並根據 GitHub 上的指導文檔進行了配置，將其打包為 ZIP 文件後成功上傳並安裝。隨後，測試者找到了木馬的路徑，並成功獲得了一個 webshell，這使得其能夠上傳一個 C2（Command and Control）服務器。在進一步的操作中，測試者嘗試使用 sudo 命令獲取 root 權限，並成功執行了相關命令。隨後，測試者發現宿主機的某些目錄被映射到容器內部，這使得其能夠進行更深入的權限提升操作。透過發現 limesvc 的密碼，測試者嘗試 SSH 進入宿主機並成功登入，這為後續的權限提升奠定了基礎。最終，測試者在宿主機上執行了一個 SUID 的 bash 腳本，並成功獲得了 root 權限。
