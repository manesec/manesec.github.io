首先，通過 rmg 工具對 jmxrmi 進行枚舉，發現系統中僅有 jmxrmi 開啟。接著，使用 beanshooter 工具針對 jmx 協議進行操作，成功枚舉到兩個用戶，但 SSH 登錄失敗，因為該系統僅支持使用 SSH 密鑰登錄。為了進一步測試，查閱文檔後發現 stand 功能可以執行任意命令，於是嘗試執行命令並成功上傳了一個 shell。隨後，利用 diagnostic read 功能讀取了系統中的任意文件，並連接到 C2 伺服器，查看 /etc/passwd 文件，發現一個名為 useradmin 的用戶。根據之前的枚舉結果，嘗試使用已知的密碼之一進行登錄，成功後卻遇到需要驗證碼的問題。為了獲取驗證碼，使用 linpeas 工具檢查系統，發現 /home/useradmin 目錄下有一個 backups 文件夾，其中的 backup.tar.gz 文件具有讀取權限。解壓後，發現了 .google_authenticator 文件，裡面包含了一個 OTP 生成密鑰。通過在線工具生成 OTP，成功獲得驗證碼並登錄系統。進一步檢查 sudo 權限後，發現可以使用 root 身份來增加用戶。由於系統是 Ubuntu，根據默認的 sudoers 文件，admin 組的用戶可以使用 sudo 獲得 root 權限。利用這一漏洞，創建一個名為 admin 的用戶，隨後切換到該用戶並使用 sudo -s 命令獲得 root 權限，最終成功獲取了系統的完全控制權限。
