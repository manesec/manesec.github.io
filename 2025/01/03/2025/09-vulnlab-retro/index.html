manesec
2025-01-03

​​

https://api.vulnlab.com/api/v1/share?id=5e345532-aedf-47d3-b2fe-8d6f19af90a9

首先使用 nmap 掃描識別目標系統為 Active Directory 環境。隨後，通過 guest 用戶訪問 SMB 共享目錄，發現了名為 Trainees 的目錄可以讀取。利用 rid-brute 技術枚舉用戶帳戶後，發現了一個名為 Important.txt 的文件，文件中透露了用戶密碼政策的線索。通過密碼噴射技術，成功獲得了 trainee 用戶的密碼，並進一步訪問了新增的 Notes 共享目錄。該目錄中包含有關舊版銀行軟件和計算機帳戶配置的提示，暗示需要從 LDAP 中查找屬性為 userAccountControl=4128 的計算機帳戶。隨後，嘗試使用 Kerberos 驗證，因為 NTLM 驗證失敗，並發現了 Active Directory Certificate Services (ADCS)。請求了一個票據後，使用 certipy 工具查找可利用的證書。由於緩存服務器問題，certipy 無法找到正確的 DC IP，因此需要指定 -dc-ip 和 -target 屬性。最終，certipy 標記了可以利用的 ESC1 證書，並在設置 -key-size 4096 後成功請求到管理員證書。通過獲取的 NTLM hash，使用 pass-the-hash 攻擊成功獲得系統控制權。
