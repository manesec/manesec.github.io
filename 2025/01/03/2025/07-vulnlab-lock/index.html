80 端口似乎沒有實質內容，而在 3000 端口中，歷史提交記錄顯示出一個 token，這使得可以訪問原始文件並下載。嘗試在本地運行後，發現用戶還有一個名為 “website” 的項目，並且可以使用該 token 進行 git clone 操作。通過 Visual Studio Code 檢查修改內容後，得知網站會自動同步項目變更，這使得可以隨意修改 changelog.txt 文件，並成功提交更改。當再次訪問時，確認修改已生效。進一步檢查服務器，發現其為 IIS 服務器，並上傳了一個 ASPX 的 shell，從而獲得了 webshell。隨後，進入 C:\gitea\data 目錄，發現了 gitea.db 數據庫文件，下載後嘗試利用 hashcat 破解 hash，但未能成功。於是，改變管理員密碼並成功登錄，然而發現內部並無有用信息。在 C:\users 目錄下檢查時，發現了一個 config.xml 文件，該文件指向 mRemoteNG 軟件，經過解密後獲得了 Gale.Dekarios 的密碼，並成功使用 RDP 進行連接。連接後發現 PDF24 軟件，並尋找相關漏洞，最終找到了一個本地特權提升的漏洞，通過尋找原始安裝文件並利用特定的步驟，最終獲得了 NT SYSTEM 權限，進而提取到了 root.txt 文件。
